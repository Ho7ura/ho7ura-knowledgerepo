# 摄像头问题诊断报告

> 日期：2026-05-27
> 硬件：Seeed Studio XIAO ESP32S3 Sense + OV3660 摄像头模块
> 固件：`compile/src/main.cpp` (PlatformIO, Arduino framework)

## 串口输出关键信息

```
[WiFi] connecting.... OK 172.20.10.2
E (1693) camera: Detected camera not supported.
E (1694) camera: Camera probe failed with error 0x106(ESP_ERR_NOT_SUPPORTED)
[CAM] init failed, reboot...
```

无限重启循环，每次都是同样的错误。

## 错误码解读

`0x106 = ESP_ERR_NOT_SUPPORTED`

`esp_camera_init()` 内部流程：

1. **CAMERA_ENABLE_OUT_CLOCK** — 开启 XCLK（GPIO10, 20MHz）
2. **SCCB_Init(40, 39)** — 初始化摄像头 I2C 总线（GPIO40=SDA, GPIO39=SCL）
3. **SCCB_Probe()** — 扫描已知传感器地址，**找到了一个设备**（有 ACK 应答）
4. 遍历 `g_sensors[]` 中 14 个已知传感器，依次调用 detect 函数 → **全部失败**
5. 返回 `ESP_ERR_NOT_SUPPORTED`

**关键事实**：如果 SCCB_Probe 找不到任何设备，错误码是 `ESP_ERR_NOT_FOUND`。现在是 `ESP_ERR_NOT_SUPPORTED`，说明 I2C 总线上有设备应答，但读取传感器 PID 寄存器的值与所有已知型号都不匹配。

## 根因分析

| 概率 | 原因 | 依据 |
|------|------|------|
| 最高 | FPC 排线接触不良 — 金手指氧化或未插紧，I2C ACK 单字节能过，但多字节寄存器读时数据位翻转 | 诊断固件用 Wire 库扫描也找不到设备，说明 I2C 信号质量差 |
| 较高 | 摄像头模块实际型号不是 OV3660（可能是 OV2640 或其它变体），且模块同时存在硬件问题 | XIAO ESP32S3 Sense 官方默认配 OV2640，该型号也在库的检测列表中但仍然失败 |
| 中等 | 摄像头模块硬件损坏 — 传感器内部故障，能响应 I2C 地址但寄存器不可读 | — |
| 较低 | PWDN/RESET 引脚配置为 -1（不控制），部分模块需要主动拉低 PWDN 才能退出省电模式 | `camera_pins.h` 中 `PWDN_GPIO_NUM = -1`, `RESET_GPIO_NUM = -1` |

## 已排除的可能性

- **传感器驱动缺失** — OV3660 驱动已编译在预编译库 `libespressif__esp32-camera.a` 中（用 `xtensa-esp-elf-nm` 验证过 `esp32_camera_ov3660_detect` (T) 符号存在）
- **build_flags 问题** — `platformio.ini` 中已添加 `CONFIG_OV3660_SUPPORT=1` 等宏，但对预编译库无效（库是 `.a` 文件，不参与重新编译）
- **ESP32 其他硬件** — WiFi 正常连接，SPI IMU、I2S 音频均无冲突
- **供电问题** — ESP32 正常运行，不是供电不足导致

## 解决方案（按顺序尝试）

### 1. 重新处理 FPC 排线

- 拔下摄像头 FPC 排线
- 用**酒精**（非橡皮擦）擦拭排线两端金手指
- **排线方向反过来**试一下（金属触点面朝向）
- 黑色卡扣压到底，确保排线完全插入
- 换一个供电充足的 USB 口给 XIAO 供电

### 2. 确认模块真实型号

- 目视检查摄像头模块上的丝印标记
- XIAO ESP32S3 Sense 默认配 **OV2640**，确认你的模块到底是不是 OV3660

### 3. 交叉验证

- 换一个确定型号完好的 OV2640/OV3660 模块测试
- 或者把当前摄像头模块接到另一个 ESP32-CAM 上验证

### 4. 临时绕过方案

用 PC 的 USB 摄像头 + `local_cam.py` 模拟 ESP32 摄像头：

```bash
python app_main.py          # 启动服务器
python local_cam.py         # 启动本地摄像头推流
```

浏览器打开 `http://localhost:8081/` 即可看到画面。整套 AI 导航功能在服务器端正常运行，只是画面来源从 ESP32 换成了 PC 摄像头。
