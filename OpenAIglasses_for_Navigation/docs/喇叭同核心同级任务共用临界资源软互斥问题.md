## 1. I2S 输出（喇叭）—— 两个任务共用

`taskHttpPlay` (Core 0) 和 `taskTTSPlay` (Core 0) 都调用 `i2sOut.write()`：

```cpp
taskHttpPlay (Core 0):    i2sOut.write(...)  // HTTP WAV 流播放
taskTTSPlay (Core 0):     i2sOut.write(...)  // TTS 分片播放
```

两个任务同核（Core 0）、同优先级（prio 2），按时间片轮转调度。如果 TTS 真正启用，两个任务交替往同一个 I2S 外设写数据，输出就是两种音频叠加的噪音。

当前靠 `tts_playing` 和 `http_play_running` 两个全局 flag 互斥规避，但没有原子性保证。



taskHttpPlay 操作喇叭
compile.ino:664-667


while (off < bytes && http_play_running) {
    size_t wrote = i2sOut.write((uint8_t*)outLR + off, bytes - off);
    if (wrote == 0) vTaskDelay(pdMS_TO_TICKS(1));
    else off += wrote;
}
taskTTSPlay 操作同一个喇叭
compile.ino:716-718


size_t wrote = i2sOut.write((uint8_t*)stereo32Buf + off, bytes - off);
if (wrote == 0) vTaskDelay(pdMS_TO_TICKS(1)); else off += wrote;
互斥机制
两边各靠一个全局 flag 做"软互斥"：

compile.ino:465 — http_wav 看 http_play_running


static volatile bool http_play_running = false;
compile.ino:87 — tts_play 看 tts_playing


volatile bool tts_playing = false;
compile.ino:698


if (!tts_playing){ vTaskDelay(pdMS_TO_TICKS(5)); continue; }
问题在哪
两个 flag 互斥完全靠"人的约定"——我设我的你不设你的。没有任何机制保证它们不会同时为 true。一旦同时为 true，两个任务同核同优先级，时间片轮流写 i2sOut，喇叭输出垃圾。

I2S 硬件驱动内部有 DMA 缓冲区，两个任务交替往里塞不同来源的数据，DMA 不管你是哪个任务来的，照单全发，结果就是两段音频碎片混在一起
